# Mail-in-a-Box in Docker

**Run Mail-in-a-Box inside a Docker container** — portable, reproducible, and isolated from the host OS.

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

---

## What this project is

[Mail-in-a-Box](https://github.com/mail-in-a-box/mailinabox) is a one-click email server that turns a fresh Ubuntu machine into a fully functional mail server. It bundles Postfix, Dovecot, nginx, Roundcube, DKIM, DMARC, SPF, NSD, BIND9, SpamAssassin, and more into a single coherent setup.

This project wraps Mail-in-a-Box v68 inside a Docker container, giving you the same integrated mail platform without dedicating an entire host to it. You can run other services alongside your mail server, update components independently, and migrate between hosts more easily.

## Why run Mail-in-a-Box in Docker?

- **Portability** — Move your mail server between VPS providers or bare-metal hosts without reinstalling from scratch
- **Coexistence** — Run web apps, databases, or other containers on the same machine without port conflicts
- **Reproducibility** — The same image builds the same mail server every time
- **Isolation** — Mail services live inside a container; host breakage won't take down email
- **Agencies & MSPs** — Standardise mail deployments across client infrastructure

## Features

- Full Mail-in-a-Box v68 installation (Postfix, Dovecot, Roundcube webmail, admin panel)
- SMTP (port 25), Submission (587), SMTPS (465), IMAPS (993), POP3S (995)
- DKIM signing, DMARC verification, SPF enforcement
- Spam filtering via SpamAssassin + Postgrey
- Let's Encrypt SSL certificate provisioning
- Authoritative DNS (NSD) and recursive DNS (BIND9) inside the container
- SNIProxy on the host for HTTP/HTTPS coexistence
- Web-based admin panel for mailbox and domain management

## Quickstart

```bash
git clone https://github.com/rasrobo/mail-in-a-docker
cd mail-in-a-docker

# Set your mail hostname
echo "PRIMARY_HOSTNAME=mail.example.com" >> .env

# Build and start
cp .env.example .env
docker compose build
docker compose up -d
docker exec miad bash /install.sh mail.example.com
```

Full instructions → [INSTALL.md](INSTALL.md)

## Prerequisites

- **A Linux server (bare metal or VPS)** with a public IPv4 address
- **Ubuntu 22.04 (Jammy)** on the host — required by Mail-in-a-Box. Other versions may work but are untested
- **Official Docker** installed (not the snap version). See [setup instructions](INSTALL.md#docker-installation)
- **DNS records** configured to point at your server (see [DNS requirements](#dns-requirements))

## DNS requirements

Before the mail server can send or receive mail, configure these DNS records at your domain registrar:

| Record | Value |
|---|---|
| `A` | `mail.example.com` → your server IP |
| `MX` | `example.com` → `10 mail.example.com.` |
| `TXT` | `example.com` → `v=spf1 mx a:mail.example.com -all` |
| `TXT` | `_dmarc.example.com` → `v=DMARC1; p=reject; rua=mailto:admin@example.com` |
| `TXT` | `mail._domainkey.example.com` → (generated by MIAB, shown in admin panel) |
| `PTR` | Your server IP → `mail.example.com` (set via VPS provider) |

## Port reference

| Port | Service | Notes |
|---|---|---|
| 25 | SMTP (inbound) | Required for receiving mail |
| 53 | DNS (authoritative) | Serves DNS for your domain |
| 80 | HTTP → 810 | Admin panel & Let's Encrypt |
| 443 | HTTPS → 4413 | Admin panel & Roundcube |
| 465 | SMTPS | Outbound submission |
| 587 | Submission | Outbound submission |
| 993 | IMAPS | Mail client access |
| 995 | POP3S | Mail client access |

## Host services

These components run on the host (outside the container):

- **Docker** — container runtime
- **SNIProxy** — routes ports 80/443 to the container's 810/4413 based on hostname
- **BIND9 + NSD** are installed on the host as part of the setup process.
  Inside the container, MIAB manages its own DNS instances.

## Compatibility

| Component | Status |
|---|---|
| Mail-in-a-Box v68 | Supported |
| Ubuntu 22.04 (Jammy) | Supported (host OS requirement) |
| Ubuntu 24.04+ | Untested — may require patches |
| Docker CE (official) | Required |
| Docker (snap) | Not supported |
| Other mail stacks | Do not run alongside MIAB on the same host |

## Known limitations

- **Beta status** — This project is functional but not yet battle-hardened for production at scale
- **DNS sync** — MIAB DNS changes are not reflected until `setup-dns.sh` re-runs on the host (see [TROUBLESHOOTING.md](TROUBLESHOOTING.md))
- **Ubuntu version** — MIAB targets Ubuntu 22.04. Running on newer releases may require manual patches
- **No Nextcloud** — Nextcloud installation is disabled in this container build
- **Firewall** — Mail-in-a-Box disables the host firewall (UFW) during setup; manage firewall rules at the host level
